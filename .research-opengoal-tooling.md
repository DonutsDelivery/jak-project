## Part 2: Tools we're NOT using that we should be

### Scenario: "I need to rebuild fast and iterate on ONE .gc file"

- **`(ml "filename")`** — REPL command (alias of `compile_load`). Compiles AND sends the object file to the live running gk; replaces in place. No restart. This is the standard Naughty Dog hot-reload loop. See `/home/user/Programs/Jak-X/jak-project/common/repl/repl_wrapper.cpp:170`.
- **`(m "filename")`** — Compile only, don't send. Good for syntax-checking a single file without a running target.
- **`(reload)`** — re-evaluate the startup file. Useful when you've edited `goal_src/user/<you>/user.gc`.
- **`(mng)`** — "make-group engine" — builds just the engine group instead of the whole iso.
- **`startup.gc` with `og:run-below-on-listen` comment** — Any lines below that magic comment re-run automatically after every successful `(lt)`. This is the entire "rebuild + reconnect + resume" loop. See `/home/user/Programs/Jak-X/jak-project/goal_src/user/README.md`.
- Example jakx startup flow (put in `goal_src/user/<name>/startup.gc`):
  ```
  (mi)
  (lt)
  ;; og:run-below-on-listen
  (dbgc)
  ```

### Scenario: "I need to inspect runtime state on a live process"

These are all REPL forms (after `(lt)`):
- **`(:bp <addr-or-symbol>)`** — set breakpoint. `(:ubp ...)` to remove.
- **`(:di)`** — "debugger info": registers, rip, backtrace.
- **`(:disasm <addr> <len>)`** — disassemble live x86 memory at a GOAL address.
- **`(:pm <addr> <size>)`** — print memory (hex dump of live process memory).
- **`(:sym-name <offset>)`** — reverse lookup a symbol from its s7-relative offset (handy when you have only a register value at a break).
- **`(:dump-all-mem)`** — dumps the entire EE process memory to disk. Use this when the game crashes and you want the state to analyze offline.
- **`(:break)` / `(:cont) / (:stop)`** — halt, continue, kill target.
- **`(dbg)` / `(dbgc)`** — attach debugger / attach-and-continue.
- **`(dbs)`** — debug status (shows what the debugger is attached to).
- All registered in `/home/user/Programs/Jak-X/jak-project/goalc/compiler/compilation/Atoms.cpp:190-204`.
- Debugger implementation + breakpoint internals: `/home/user/Programs/Jak-X/jak-project/goalc/debugger/Debugger.h`.

### Scenario: "I want to introspect GOAL types on a live process"

- **`(print-type <expr>)`** — type of any expression in the REPL.
- **`(method-of-type <type> <method-name>)` / `(method-of-object ...)`** — resolve a vtable slot.
- **`(size-of <type>)` / `(psize-of ...)` / `(offset-of ...)`** — sizeof / offsetof from the REPL.
- These all work against the **live compiler's type system**, so you can probe `all-types.gc` without booting.

### Scenario: "The ported file compiles but looks wrong"

- **`tools/type_searcher`** — find types by size/parent/field layout. Invocation:
  ```
  build/tools/type_searcher --output-path ./search-results.json --game jakx \
    --fields '[{"type":"int16","offset":2}]' --size 128 --parent process
  ```
  Full flags at `/home/user/Programs/Jak-X/jak-project/tools/type_searcher/main.cpp:70-100`. Emits JSON of all matching types. Hugely underused for "what type is at this address?"
- `task type-search` — pre-wired invocation via Taskfile.

### Scenario: "I want to know what the original game had at a memory address"

- **`tools/memory_dump_tool`** — parses a PCSX2 `eeMemory.bin` with full knowledge of `all-types.gc`. Dumps the heap with type annotations. One-shot:
  ```
  build/tools/memory_dump_tool ./savestate_out/eeMemory.bin \
    --output-path ./ --game jakx > ee-analysis.log
  ```
- **`scripts/memory-dump-p2s.py`** — unzips a `.p2s` PCSX2 savestate into an `eeMemory.bin`.
- **`task analyze-ee-memory FILE=<path to .p2s>`** — does both of the above in one command.
- **`task watch-pcsx2 SAVESTATE_DIR=<dir>`** — watches PCSX2's savestate folder; every time you save state, it re-runs the dump tool. Live comparison of ported engine state to real game state. Requires `watchmedo`.

### Scenario: "Decompilation output for one file is broken"

- **`task decomp-file FILE=<name>`** — re-run decompiler for one object only. Uses `--config-override` `"allowed_objects": ["<file>"]` so it runs in seconds. See `Taskfile.yml:132`.
- **`task update-gsrc-file FILE=<name>`** — decompile + patch your `goal_src/jakx/**/<file>.gc` from the new output, preserving comments. This is the script that's smart about comment preservation (see `/home/user/Programs/Jak-X/jak-project/scripts/gsrc/update-from-decomp.py`).
- **`task update-gsrc-glob GLOB="<pattern>"`** — same for a glob.
- **`task decomp`** — full decompile. **`task extract`** — assets only (no code).
- **`task rip-textures` / `rip-levels` / `rip-collision` / `rip-audio`** — asset extraction profiles, each with its own `--config-override`. These are the single-shot "I just want PNGs" / "I just want .glb files" / "I just want .wav" invocations.

### Scenario: "I want to validate my ported decomp matches the original output"

- **`build/offline-test`** — reference test that runs decompiler against every file with a `_REF.gc` fixture in `test/decompiler/reference/`. Flags:
  - `--file <name>` — one file only
  - `-d/--dump_current_output` — dump current output to `failures/` for comparison
  - `-t/--num_threads 8` — parallel
  - `--fail-on-cmp` — hard fail on mismatch
  - `-p/--pretty-print` — progress UI
- **`task offline-test-file FILE=<name>`** — wrapper for single file.
- **`task update-ref-tests`** — updates ALL `_REF.gc` fixtures from current decompiler output. Loop: run offline-test with `--dump_current_output`, feed failures through `scripts/update_decomp_reference.py`, re-run.
- **`task update-ref-file FILE=<name>`** — same for one file.
- These are meta-valuable for jakx: once you've stabilized a file's decomp, drop a `_REF.gc` next to it to catch regressions when you retune `type_casts.jsonc`/`hacks.jsonc`.

## Part 3: Runtime flags on `gk` (complete list)

From `/home/user/Programs/Jak-X/jak-project/game/main.cpp` (gk-level) and `/home/user/Programs/Jak-X/jak-project/game/kernel/jakx/kmachine.cpp:57-218` (game-level, passed after `--`).

**gk-level flags (before `--`):**
- `--version` — print revision hash and exit.
- `-g,--game <name>` — `jak1|jak2|jak3|jakx`.
- `-v,--verbose` — debug-level stdout logging.
- `--port <N>` — listener port; default 8112 + version.
- `--no-avx2` — force-disable AVX2 (test non-AVX2 code paths).
- `--no-display` — headless (no window). Critical for CI/automated smoke tests.
- `--profile` — start profiler immediately.
- `--profile-until-event <name>` — stop profiler at event; dumps JSON.
- `--gpu-test opengl --gpu-test-out-path <file>` — GPU compat test, no boot.
- `--portable` — store saves/settings next to the exe (mod distribution).
- `--config-path <dir>` — relocate user config dir.
- `--disable_save_location_override` — pairs with `--config-path`; keep saves in default location.
- `--proj-path <dir>` — override location of `data/`.
- `--disable-ansi` — no color codes in logs.

**Game-level flags (after `--`, parsed by kmachine.cpp for jakx):**

ISO source:
- `-cd` — use real DVD drive (retail).
- `-cd-local-overlord` — DVD drive but local overlord.irx. (jakx-specific, mostly stubbed.)
- `-cddata` — DVD for data, local for IOP modules.
- `-deviso` — dev-ISO mode. Jak X specifically **does not remove this**, despite the game_arg_documentation claiming it was removed.
- `-fakeiso` — no DVD, read raw files from `out/jakx/iso/`. **This is the default for dev.**

Boot mode:
- `-boot` — retail mode (no debug segments, auto-boot the game).
- `-debug-boot` — retail mode but WITH debug segments loaded.
- `-debug` — dev mode (debug + debug segments).
- `-debug-mem` — debug mode but skip loading debug segments.
- `-nokernel` — don't load KERNEL.CGO. Useful for pure runtime testing.
- `-nosound` — mute overlord audio.

Scene selection:
- `-level <name> <symbol-id>` — boot directly into a level. **WARNING**: jakx kmachine.cpp:184 has `ASSERT_NOT_REACHED()` on the symbol-id path — this flag currently crashes.
- `-user <name>` — debug user name (for conditional code paths).
- `-art <name>` — boot directly to an art-group (useful for viewing individual actors).
- `-demo` / `-kiosk` / `-preview` — demo-mode variants.
- **`-beta` — jakx-specific**, sets DebugBootMessage to "beta". (kmachine.cpp:97-101)

Debug symbols:
- `-debug-symbols` / `-no-debug-symbols` — toggle DebugSymbols flag.

Jak X does NOT yet wire:
- `-overlord` / `-overlord2` (commented out at kmachine.cpp:157-165, pending overlord2 port).

## Part 4: Decompiler config options worth experimenting with

Our `decompiler/config/jakx/jakx_config.jsonc` is 201 lines. Many flags are **off** that could surface more data. Full option list from `/home/user/Programs/Jak-X/jak-project/decompiler/config.h:110-194`:

### High-value to flip ON temporarily when debugging 305 failing files:

- **`"disassemble_code": true`** — generate plain MIPS .asm for every function. Guaranteed 100% success. Invaluable for "what did the original code actually do" when decompilation fails.
- **`"hexdump_code": true` / `"hexdump_data": true`** — hex dumps inline in the output. Good for validating patches.
- **`"print_cfgs": true`** — control flow graph per function. Debug tool for atomic-op/CFG crashes (we hit this on all the wvehicle-*, race-line, etc. in banned_objects).
- **`"write_hex_near_instructions": true`** — show raw bytes beside decoded instructions. Catches instruction-decoder bugs.
- **`"dump_objs": true`** (already on) — writes raw object files to decompiler_out for external inspection.

### Per-run override via `--config-override` (no config file edit needed):

```
build/decompiler/decompiler decompiler/config/jakx/jakx_config.jsonc \
  iso_data decompiler_out --version ntsc_v1 \
  --config-override '{"allowed_objects":["wvehicle"],"print_cfgs":true,"disassemble_code":true}'
```

This is the pattern to ship to the wvehicle crashes: restrict to one object, dump everything the decompiler can tell us, before vs after.

### For stabilizing types:

- **`"generate_all_types": true`** — regenerate `all-types.gc` from scratch based on current function signatures. Run once with all objects allowed, diff the output, merge manually.
- **`"generate_symbol_definition_map": true`** — emits `symbol_map.json`. Guesses which object file first-defines each symbol. Useful when porting jak3 engine code — tells you where a type is "supposed" to come from.
- **`"dump_art_group_info": true` / `"dump_joint_geo_info": true` / `"dump_tex_info": true`** — dumps JSON catalogs (already pointed at in the config but the *process* flags are false). Run with all objects allowed.

### Already on, good:

- `apply_patches: true`, `read_spools: true`, `find_functions: true`, `levels_extract: true`, `extract_collision: true`, `write_scripts: true`, `dump_objs: true`.

### Turned off deliberately, leave alone:

- `is_pal` — we're ntsc_v1. The PAL override is in `version_overrides.pal`.
- `write_patches` — only for creating new xdelta3 patches against specific objects. Off unless creating a new patch.

### Per-file tuning files (one per language-file; under-edited for jakx):

These JSONC files in `decompiler/config/jakx/ntsc_v1/` let you fix individual function signatures without touching the main config:
- `type_casts.jsonc` — cast a register at a specific op-idx. Rescues functions whose type inference fails.
- `var_names.jsonc` — rename `a1-0` → something useful.
- `label_types.jsonc` — declare a label is `(pointer foo)` when the decompiler can't tell.
- `stack_structures.jsonc` — tell the decompiler what's on the stack.
- `hacks.jsonc` — bag of escape hatches: `no_type_analysis_functions_by_name`, `mips2c_functions_by_name` (emit C instead of GOAL for a function!), `reject_cond_to_value`, etc.
- `anonymous_function_types.jsonc` — types for lambdas-without-names.

**Particularly useful jakx entry point**: `hacks.jsonc` → `mips2c_functions_by_name`. When a single inline-asm function sinks a file, declare it mips2c and the rest of the file decomps cleanly. The C output goes in `game/mips2c/`.

## Part 5: Gotchas / things that look like tools but don't work

- **`scripts/shell/decomp.sh` does NOT run jakx decomp.** It hardcodes jak1. Use `task decomp` (which reads the game env set by `task set-game-jakx`).
- **`-level <name> <id>` in jakx crashes** — `ASSERT_NOT_REACHED()` at `kmachine.cpp:184`. Don't use it until someone ports the symbol-ID decoder.
- **`-overlord` / `-overlord2` are disabled in jakx** — commented-out in `kmachine.cpp:157-165`. Jak X runs overlord1 only, no flag switches this.
- **`tools/level_tools/level_dump`** is hard-coded to `GameVersion::Jak1` (`constexpr GameVersion kGameVersion = GameVersion::Jak1;` at `/home/user/Programs/Jak-X/jak-project/tools/level_tools/level_dump/main.cpp:13`). Needs a one-line change to work for jakx.
- **`tools/type_searcher` only supports jak1/2/3 in its switch statement, with a fallthrough bug**: `/home/user/Programs/Jak-X/jak-project/tools/type_searcher/main.cpp:110-113` — JakX has no `break;` before `default:`, so it falls through to "unsupported". Needs a break inserted, then works.
- **`scripts/gsrc/skeleton_creation/init_folder_struct_jakx.py` does not exist.** Cloning the jak3 version is a one-hour chore that would have saved hundreds of manual file moves.
- **`scripts/batch/*jakx*.bat` does not exist.** None of the Windows one-shot scripts have jakx variants.
- **`offline-test` does not currently have jakx reference files** under `test/decompiler/reference/`. Until you add `_REF.gc` fixtures, `task offline-tests` for jakx reports zero coverage but also zero failures — it's misleading.
- **`decompiler/config/jakx/all-types.gc.agent2-broken.bak` and `all-types.gc.bak`** exist alongside the live file. These are backups, not magic versions. Don't read them expecting a cleaner baseline.
- **`collect_errors.sh` mutates `goal_src/` in place** (writes empty files over failing .gc, then restores from `.orig`). If it crashes midway, run the restore loop manually or you'll lose code. Consider wrapping in `git stash` safety. The script's output `/tmp/jakx_all_errors.txt` is the useful artifact — the in-place modifications are a hack.
- **`--config-path` + `--portable` precedence**: `--portable` wins. If you set both, your `--config-path` is ignored.
- **nREPL port collision**: goalc's nREPL defaults to 8181. If you launch two goalc instances, the second silently fails to bind.

## Surprising finds worth calling out

1. **`og:run-below-on-listen` startup magic** (repl.md convention) — "rebuild + reconnect + resume after crash" is a one-line convention, not a tool. Put `;; og:run-below-on-listen` in your startup.gc and every `(lt)` re-runs everything below it. This is the loop everyone should be using for crash-recovery iteration.
2. **`task watch-pcsx2`** — live-compare PCSX2 savestates against our ported engine's type definitions. Every time you save state in PCSX2, it re-dumps the analyzed heap. This is the best validation tool for "does my ported type layout match the original?"
3. **`mips2c_functions_by_name` in `hacks.jsonc`** — declare a function too hard to decompile, and it's machine-translated MIPS → C instead. Full escape hatch; the output lives in `game/mips2c/` and links into the runtime. We should use this aggressively on the wvehicle-physics / wvehicle-ai functions that currently break `atomic_op`.
4. **`-debug-boot` vs `-debug` vs `-debug-mem`** — three distinct debug modes. `-debug-boot` = retail auto-boot + debug segments loaded (closest to "shipping with dev tools"). We've been assuming `-debug` is the only debug mode; `-debug-boot` is what you want when you're past bring-up and testing gameplay.
5. **`goalc -c "<form>"` one-shot mode** — compiler runs the form and exits. CI can verify "does this specific file still compile?" in ~2 seconds without the REPL. Underused for scripted validation.
6. **ImGui "Reboot in Debug Mode" menu item** — `Debugging > Reboot In Debug Mode!` triggers `RESTART_IN_DEBUG` which re-launches gk with `-boot -debug` appended. No terminal juggling. Set near line 111 of debug_gui.cpp.
