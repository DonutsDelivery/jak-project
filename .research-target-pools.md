## Definition Details (jakx == jak3, byte-identical)

```lisp
;; gkernel.gc:112
(defconstant *kernel-dram-stack* (&+ *dram-stack* DPROCESS_STACK_SIZE))

;; gkernel.gc:2349
(define *target-dead-pool*
  (new 'global 'dead-pool 1 #x16800 "*target-dead-pool*"))
;; -> 1 slot of 92 KB (0x16800 = 92,160 bytes). One target fits.

;; gkernel.gc:2382-2383
(change-parent
  (define *target-pool* (new 'global 'process-tree "target-pool"))
  *active-pool*)
(set! (-> *target-pool* mask)
  (process-mask freeze pause menu progress process-tree))
```

`*target-dead-pool*` is also registered in `*dead-pool-list*` (line 2363) for GC/debug walks.

## Runtime Loadedness Confirmation

`goal_src/jakx/game.gp:82` compiles `gkernel.o` into KERNEL.CGO:
```
(cgo-file "game.gd" '("$OUT/obj/gcommon.o" "$OUT/obj/gstate.o"
                      "$OUT/obj/gstring.o" "$OUT/obj/gkernel.o"))
```
KERNEL.CGO loads during boot before any game code runs, so all three symbols are initialized at runtime by the time `start` would execute.

## Usage Evidence in jakx

The same pool trio is already consumed elsewhere in jakx without issue:
- `engine/game/game-save.gc:2971-2972` — `:to *target-pool* :stack *kernel-dram-stack*`
- `engine/level/level.gc:2938` — `:stack *kernel-dram-stack*`
- `engine/game/main.gc:87` — `:stack *kernel-dram-stack*`
- `engine/debug/memory-usage.gc:118` — reads `*target-dead-pool*`
- `engine/gfx/hw/display.gc:79` — writes to `*target-pool*`

## Recommendation

**Safe to port `start` as-is.** No pool creation work required. All three referenced symbols (`*target-dead-pool*`, `*target-pool*`, `*kernel-dram-stack*`) are defined in jakx at the same locations and with the same sizes as jak3, compiled into KERNEL.CGO, and already in use by other jakx code. The 92 KB single-slot target-dead-pool is appropriate — jakx only spawns one `target` process at a time, same as jak3.

No `(define ...)` additions needed.
