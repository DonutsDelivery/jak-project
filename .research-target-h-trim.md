## Concerns

- **Method-number drift**: current target-h lists `(target-method-28 ...)` which collides with process-focusable's slot. Must delete.
- **State-number drift**: states are by name not number in OpenGOAL — no drift, but any downstream code that did `(go target-walk)` will break (already broken at runtime if ever reached).
- **vehicle-spawn** comment header is "target-handler" in current file; actual definition is vehicle-manager. Just relocate the `;;` header comment.
- **sidekick deftype** (lines 593-609): keep, it's used by the inspect output and is referenced widely. Just verify `(:states sidekick-clone)` — grep: zero hits in jakx — REMOVE sidekick-clone state.
- **darkjak-stage / lightjak-stage / target-flags / target-geo / target-anim / target-effect enums**: keep all, they're type definitions not link symbols. Though most bits/values are now unused, removing them is cosmetic and risks breaking field types.
- **target struct fields**: current port lists ~100 fields (racer/tube/flut/board/pilot/gun/mech/turret/indax/ladder/darkjak/lightjak). Inspect output only prints fields at offsets 232,236,252,260,268,276,284,292,296,300,316,332,348,396,404,412,420,428,436,444,452,460,468,476,484,492,500,504,508,516,524,532,540,548,552,556,572,588,684,716,732,764 ending at size 0x350=848. Actual field list is ~40 fields, not 100. A follow-up task should reconcile the struct layout — current oversize fields produce wrong offsets and will corrupt memory the moment anything writes to `target->racer` etc.
